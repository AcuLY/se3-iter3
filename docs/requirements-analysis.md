# 需求分析文档

项目名称：Journey Skill Agent 旅行风格智能体工作台  
迭代阶段：迭代三  
日期：2026-06-14

## 1. 项目背景

迭代三要求把 Agent 能力从“可演示的对话能力”推进到“可运行、可评估、可复盘的产品能力”。本项目确定的产品主线是：

**旅行风格 Skill 分享平台 + 多 Agent 行程规划工作台。**

系统不只输出一段 Markdown 行程，而是把 Agent 结果写入可编辑的结构化行程画布。用户可以完全手动规划，也可以导入旅行风格 Skill 后让 Agent 继续补全、检查和改写。

## 2. 用户角色

- 普通旅行用户：创建和编辑行程，导入 Skill，让 Agent 协助补全或优化行程。
- Skill 创作者：从行程、对话或外部游记中提取旅行风格 Skill，编辑确认后发布。
- 答辩/开发人员：查看评估后台、Bad Case、优化前后指标和 Agent trace 摘要。

## 3. 核心业务闭环

1. 用户创建旅行行程，填写目的地、日期、同行人和偏好。
2. 用户可在中间画布中手动添加、编辑、删除活动。
3. 用户可从 Skill 广场导入旅行风格 Skill。
4. 右侧 Agent 根据当前行程、用户输入和导入 Skill 更新画布。
5. 系统在右侧对话的本轮 Agent 回复末尾展示 diff，让用户知道 Agent 改了什么；用户手动编辑不计入该 diff。
6. 用户可从已有行程或外部文本提取 Skill 草稿。
7. 用户确认后发布 Skill，其他用户可推荐、收藏、导入。
8. 评估后台展示优化前后指标，支撑 Agent 优化文档和答辩。

## 4. 功能需求

### 4.1 首页

- 展示产品名称、定位和核心入口。
- 提供进入工作台、创作 Skill、Skill 广场入口。
- 首页保持较轻的产品介绍，不堆叠复杂功能。

### 4.2 行程工作台

- 左侧为 ChatGPT 式导航栏，上方为功能入口，下方为会话记录。
- 中间为主要行程编辑画布。
- 顶部展示地图区域，显示目的地、活动点和路线摘要。
- 中下部展示横向日期导航和纵向活动列表。
- 支持用户不依赖 Agent 完整手动规划：
  - 添加日期。
  - 添加活动。
  - 删除活动。
  - 编辑活动标题、地点、开始时间、结束时间、说明、预算、交通备注。
  - 展示住宿、餐饮、交通、景点、自由活动等活动类型。

### 4.3 Agent 对话

- 右侧为 IDE 风格 Agent 对话栏。
- 支持导入 Skill。
- 支持用户用自然语言要求补全、检查或改写行程。
- Agent 更新中间画布，而不是只输出 Markdown。
- Agent 回复末尾展示本轮 diff；用户直接在画布中的手动编辑只更新行程，不生成对话 diff。
- 普通用户界面默认不暴露完整 trace，只展示摘要。

### 4.4 Skill 广场

- 支持浏览 Skill。
- 支持根据目的地、偏好、当前行程语境推荐 Skill。
- 支持收藏 Skill。
- 支持导入 Skill 到当前行程。
- 支持发布、下架、删除等后端接口。

### 4.5 Skill 创作与提取

- Skill 使用 `skill-creator` 标准格式。
- `SKILL.md` 必须包含 YAML frontmatter，至少包含 `name` 和 `description`。
- 系统可从当前行程和外部文本中生成 Skill 草稿。
- 提取结果默认是 draft，不能直接发布，必须由用户确认。

### 4.6 地图与工具能力

- 地图能力按高德 API 方向设计。
- 当前实现提供后端统一封装和本地 fallback：
  - POI 搜索。
  - 路线规划。
  - 天气查询。
- 没有高德 Key 或外部服务失败时也能离线演示和测试。

### 4.7 评估后台

- 展示评估用例。
- 展示优化前后指标。
- 支撑答辩说明 Bad Case、优化实现和优化效果。

## 5. 非功能需求

- 可运行性：无外部 API Key 时仍能本地启动和演示。
- 可测试性：共享领域逻辑、后端 API、前端核心交互均有自动化测试。
- 可扩展性：共享类型由 `packages/shared` 统一定义，前后端复用。
- 可追踪性：Agent 调度过程记录 trace，普通 UI 默认折叠。
- 可维护性：前端页面、后端服务、共享领域逻辑分层。

## 6. 验收标准

- `npm test` 全量通过。
- `npm run typecheck` 全量通过。
- `npm run build` 全量通过。
- API 健康接口 `/api/health` 正常响应。
- 浏览器可打开 `http://localhost:5173`。
- 用户可完成“进入工作台 -> 添加活动 -> 导入 Skill -> Agent 更新行程 -> 在右侧对话末尾查看 diff”核心路径。
- 评估后台可展示优化指标。
