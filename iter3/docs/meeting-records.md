## 会议 1：迭代三方向确认

时间：2026-06-14  
参与者：项目成员、Codex  
主题：确定迭代三产品主线

### 讨论内容

- 任务 PDF 要求交付完整代码、需求分析、详细设计、会议记录、Agent 优化文档、评估数据集和答辩 PPT。
- 原有亮点“冲突检查”过于泛化，难以形成业务闭环。
- 确定业务亮点为旅行风格 Skill 分享平台。
- 确定技术亮点为主 Agent 调度子 Agent，多 Agent 上下文隔离。

### 结论

- 产品主线：旅行风格 Skill 分享平台 + 多 Agent 行程规划工作台。
- 支持用户上传、提取、编辑、发布和导入 Skill。
- Agent 输出必须更新结构化行程画布。

## 会议 2：前端布局与交互粒度

时间：2026-06-14  
参与者：项目成员、Codex  
主题：确定核心界面结构

### 讨论内容

- 旅行规划产物复杂，不能只做普通聊天页。
- 用户需要看到行程随着对话逐步完善。
- 用户必须不依赖 Agent 也能纯手动完成完整行程。
- Agent trace 不应在普通用户核心界面中过度暴露。

### 结论

- 左侧：ChatGPT 式导航栏。
- 中间：行程编辑画布，顶部地图，中下部日期导航和活动列表。
- 右侧：IDE 风格 Agent 对话栏，支持导入 Skill。
- 评估页面只作为答辩/开发后台。

## 会议 3：技术路线确认

时间：2026-06-14  
参与者：项目成员、Codex  
主题：确定技术栈和能力边界

### 讨论内容

- 迭代三使用 TypeScript 全栈。
- 前端使用 React + Vite + Tailwind。
- 组件库使用 shadcn 风格组件。
- 后端使用 Node.js + Express + SQLite。
- 地图使用高德 API 方向。
- 车票、机票只做规划辅助，不做购票、支付和实时余票保证。

### 结论

- 新建 `iter3/`，不直接沿用迭代二 Python 主线。
- Agent 框架方向为 OpenAI Agents SDK for TypeScript，当前实现保留依赖和适配点。
- 外部 API 默认有 mock fallback，保证演示稳定。
